Safety & Guardrails

4 skills
Guard
/guard
Full safety mode. Combines destructive command warnings with directory-scoped edit restrictions. Maximum protection for prod environments.
dual protection warnings + boundaries
Usage: /guard -- combines /careful + /freeze
Careful
/careful
Warns before destructive commands: rm -rf, DROP TABLE, force-push, git reset --hard, kubectl delete. User can override each warning.
hook-based overridable safe exceptions
Usage: /careful -- session-scoped
Freeze
/freeze
Lock file edits to a specific directory. Prevents accidental changes to unrelated code during debugging or scoped work.
directory scope session-scoped
Usage: /freeze src/components/
Unfreeze
/unfreeze
Clear the freeze boundary, allowing edits to all directories again. Non-destructive -- can re-freeze anytime.
clear boundary re-freezable
Usage: /unfreeze
