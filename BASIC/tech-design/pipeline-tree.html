          • 3σ 异常检测 (ATR×1.5)
            • → FC: 防守平仓触发 FC

      • 🏛️ Alpaca 全美股 — 8000只, 每周同步

          • 🔽 四层漏斗 8000→4000→500→100

              • → MC: DQN 选股范围 (100→8) MC

      • 📈 Polygon.io — 或 Yahoo Finance fallback
        • 💾 Parquet 预计算 — 季度更新
          • 基本面因子 (PE, PB, ROE, EPS)
            • → MC: 因子融合 → RL 选股 MC
      • 🏦 SEC EDGAR — 10-K/10-Q, 8-K 速报
        • 🤖 GPT-4.1 long context — 60k tokens 解析
          • 关键财务指标 JSON
            • → MC: 基本面因子补充 MC
            • → FC: 8-K 速报触发信号 FC
    • D. 新闻数据
      • 📰 NewsAPI — 70k+ 来源, ~500条/天, 免费
      • 🔍 Perplexity — AI 搜索聚合, 同级数据源, 按用量
      • ⚠️ Nasdaq RSS — 熔断/暂停/除牌, 免费
      • 🔽 G1 快速过滤 — <1s, 跨源去重 + 实体识别 + 来源检查, 95% 淘汰
        • 💾 SQLite news_events — 全量存储, g_level 分级
          • → MC: 周度情绪聚合因子 MC
          • → 训练: 新闻-收益关联训练集 Train
        • 🧠 G2 FinBERT 本地 — <5s, 零 API 成本
          • sentiment_score + importance_score + urgency
            • 🤖 G3 Claude Haiku — <30s, ≤10篇/天, importance>0.7 & urgent, <$2/月
              • 持仓影响 ±X% + target/stop 建议 (Phase 2)
            • 📊 信号评分 — score = 情绪 × 重要度 × 紧急度
              • → FC Tier-1 (>85): 自动执行 FC
              • → FC Tier-2 (60-85): Victor 审批 FC
              • → FC Tier-3 (<60): 观察列表 FC
      • 🕸️ 公司关系图 — NetworkX, 100节点, 500边 (供应链/竞争/同行业)
        • 📡 新闻传播引擎 — G2 情绪沿关系边传播
          • propagation_score (每家公司)
            • → MC: 因子融合补充 MC
            • → FC: 关联股信号放大 FC
    • E. 宏观经济数据
      • 🏛️ FRED API — 19指标, 8大维度 (利率/就业/物价/增长/信用/流动性/汇率/商品)
        • 💾 Parquet (历史) + SQLite (最新值)
          • Z-score 标准化 — 252日窗口, [-3, +3]
            • 经济周期分类 — 扩张/过热/滞胀/衰退/复苏
            • → 训练: reward shaping Train
      • 🎰 Polymarket — top 30 宏观事件概率, 免费
        • 事件概率 delta 检测 — "概率悬崖" 突变
      • ⚙️ MacroTiltEngine — 敏感度矩阵 (11 GICS × 19 指标)
        • sector_tilt (±3%) + style_tilt (价值/成长)
          • → MC: 宏观覆盖层 (Soft Overlay) MC
      • 🕸️ 宏观传播图 — NetworkX DiGraph, 30节点, 70有向边
        • 📡 冲击传播引擎 — Z>1.0 触发, impact = z × polarity × strength, 衰减0.5, 深度3
          • 行业 Tilt 补充 — 0.6 规则 + 0.4 传播
            • → MC: MacroTiltEngine 融合 MC
    • F. 内部数据 (系统产生)
      • 💼 当前持仓状态 — Alpaca 账户
        • 持仓快照 — ticker, weight, pnl, hold_days
          • → MC: 约束检查 (sector cap, position cap) MC
          • → FC: 退出管理 (TTL / target-stop) FC
          • → 决策 LLM: 当前 vs 建议组合对比 MC
      • 📋 历史交易记录 — order_id, trigger_source, signal_score
        • 💾 SQLite trades 表 → 绩效统计 — Sharpe, 胜率, 回撤
          • → 训练: RL reward 信号 Train
          • → 训练: FC 退出模型 (ML校准 + RL动态退出) Train
      • 🧬 模型权重 (.pt) — 周日训练产出 (DQN, PPO/SAC, ML校准器, RL退出模型)
        • → MC: RL 推理 (选股 + 权重) MC
        • → FC: 退出模型推理 (Phase 2) FC

汇入决策 — 三条 Chain 各自收到什么

  • 🔵 MC 决策 LLM — 周度, 周五 16:00
    • 因子融合分数 (100只 × 100D) ← A 技术因子 + C 基本面 + D 情绪 + D propagation + E MacroTilt
    • RL 推理结果 ← DQN: 8只选股 + PPO: 权重分配
    • 宏观覆盖层 ← E sector_tilt + style_tilt (±3%)
    • 当前持仓快照 ← F 持仓状态
    • 宏观环境摘要 ← E 经济周期 + Polymarket 概率变化
    • 周度新闻摘要 ← D news_events 聚合
    • 验证层 — Schema + 风控约束 + 异常检测
    • 输出: 建议组合 (8L+1S) + 权重 → Victor 审批 → Alpaca 执行
  • 🟡 FC 决策 — 实时, 新闻触发
    • G2/G3 分析结果 ← D 情绪 + 主题 + 紧急度 + 影响评估
    • 信号评分 + Tier 分级 ← D score = 情绪 × 重要度 × 紧急度
    • 传播信号 ← D 公司图 propagation_score
    • 当前持仓快照 ← F 持仓状态 + TTL/target/stop
    • 实时价格 ← A 价格监控
    • 退出模型推理 (Phase 2) ← F ML 校准 + RL hold/sell
    • 约束检查 — U100, ≤5%单笔, ≤15%总量, ≤2次/天, 30min冷却
    • 输出: Tier-1 自动执行 / Tier-2 审批 / Tier-3 观察
  • 🟣 后台训练 — 周日 10:00
    • 历史 OHLCV + Alpha158 因子 ← A (Parquet, 5-10年)
    • 历史宏观 Z-score + 周期标签 ← E (Parquet)
    • 历史交易记录 + 实现收益 ← F (SQLite)
    • FC 交易记录 (Phase 2) ← F (退出模型训练集)
    • DQN+PPO/SAC 训练 — 16h full / 2h incremental + ML校准器 + RL退出模型
    • 输出: 模型权重 (.pt) → MC + FC 共用

MC ↔ FC 交互规则

  • MC 周度重算时评估 FC 仓位
    • FC 新仓被 MC 选中 → 并入 MC, TTL/target/stop 取消
    • FC 新仓未被选中 → 按 FC 退出策略处理 (TTL 到期 / target-stop)
    • FC 调仓 MC 持仓 → MC 重新计算权重, FC 调整归位
    • 3σ 防守触发 → 立即平仓 (MC/FC 均适用, 优先级最高)
