## 7. 关键技术栈

| 模块 | 技术选型 | 理由 |
|------|--------|------|
| 数据存储 | Parquet + SQLite | 列式存储高效，同时支持关系查询 |
| LLM路由 | OpenAI + Anthropic + Google | 多模型冗余，成本优化 |
| 小模型 | FinBERT + Fin-E5 + LightGBM | 金融领域专用，推理快 |
| RL框架 | Gymnasium + PyTorch | 标准化环境，梯度优化 |
| 知识图谱 | NetworkX + CSV | 轻量内存图，毫秒级传播计算 |
| 因子引擎 | Pandas + NumExpr | 灵活表达式，高性能计算 |
| 交易API | Alpaca | 清晰API，回测友好 |
| 爬虫 | BeautifulSoup + Playwright | 解析灵活，动态渲染支持 |

|---------|---------|------|
| 数据架构 | 单元测试 + 集成 | 100% 读写正确性，P99延迟 < 500ms |
| 投资组合 | 单元 + 回测 | 约束满足100%，夏普率稳定 |
| LLM | 样本评估 | 相关性准确率 > 85% |
| 因子 | IC/ICIR评估 | IC > 0.05，单调性 > 0.7 |
| RL | 走向评估 + 实盘 | 相对基准 Sharpe +5%，MDD < 基准20% |
| 新闻 | 混淆矩阵 | 精确率 > 80%，召回率 > 70% |
| 交易 | 订单日志 | 幂等性100%，滑点 < 目标3% |
